Организация данных в Microsoft Dynamics CRM

Хотя не требуется обладать большими знаниями о базах данных, чтобы приступить к работе с Microsoft Dynamics CRM, полезно знать кое-что об организации данных в системе.

В частности, существует два определения, о которых необходимо знать, так как они используются в различных местах в системе: запись и тип записи.

Каждый день вы будете работать с различными записями клиентов и типами записей в ходе бизнес-процесса, собирая данные, необходимые для заполнения полей в их записях, и, в конечном счете, для закрытия сделки.

Что такое запись?

В Microsoft Dynamics CRMзапись — это цельная единица информации. Считайте ее одной строкой в таблице с несколькими столбцами (или полями) для хранения частей информации, составляющих всю строку.

Например, для организации можно создать столбец Название организации и Адрес, а для пользователя, которому следует позвонить при регистрации в организации, — Имя контакта. При каждом добавлении новой организации в систему создается новая запись в базе данных CRM.

Что такое тип записи?

Каждая запись, добавляемая в систему, принадлежит к определенному типу записи, например организация, контакт, интерес, возможная сделка или обращение. (В Microsoft Dynamics CRM существует несколько других типов записей помимо этих, но с этими, возможно, вы будете работать чаще всего.)

Типы записей предоставляют способ группировать и упорядочивать похожие данные. Например, в Microsoft Dynamics CRM записи контактов будут сгруппированы в плитке для типа записей контакта.

Записи обращений группируются в плитке для типа записей обращения, записи организаций в плитке для типа записей организации и т. д.
