Allow Email Router to use credentials

When Authenticate Using in the email configuration profile is set to Credentials Specified by a User or a Queue, the E-mail Router or server-side synchronization requires the credentials for the respective user or queue to process email in Microsoft Dynamics CRM. 

Allow the E-mail Router to use your credentials

  1. On the nav bar, on the top-right corner, click or tap the Settings icon > Options.

  2. In the Set Personal Options dialog box, click or tap the Email tab.

  3. Click or tap the View your Mailbox link.

  4. In the mailbox record, for Allow Server-Side Synchronization or Email Router to use Credentials, click or tap Yes, and then specify the user name and password.

Note

By default, server-side synchronization and E-mail Router require HTTPS to be used with Microsoft Dynamics CRM. If you are using HTTP for Microsoft Dynamics CRM, the Allow Server-Side Synchronization or Email Router to use Credentials field will be disabled. To enable the option, change the value of the deployment property AllowCredentialsEntryViaNonSecureChannels to 1. If you are using the Microsoft Dynamics CRM 2013 Email Router, also change the value of the Windows registry key DisableSecureDecryptionKey to 1. For more information about how to do this, see Keep user credentials secure.
