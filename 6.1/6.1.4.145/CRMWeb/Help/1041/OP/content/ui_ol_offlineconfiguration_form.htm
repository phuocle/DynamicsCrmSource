Outlook 用 CRM をオフラインで作業

Outlook 用 CRM をインストールすると、所有する Microsoft Dynamics CRM レコードを使用してオフラインで作業できます。オフライン作業中に、新しい情報を入力し、既存のレコードや活動を編集できます。オフラインでの操作方法の例については、 オフライン アクセス対応 Outlook 用 Microsoft Dynamics CRM を使用するを参照してください。

重要

初めてオフラインにするときは、ローカル データを Microsoft Dynamics CRM Server と同期するためにかなりの時間がかかることがあります。Outlook 用 CRM の同期中は、Microsoft Dynamics CRM を利用できません。同期するデータ量を制限するために、フィルターを使用することで、時間を短縮できます。

Outlook 用 Microsoft Dynamics CRM がインストールされているコンピューターにローカルにログオンするユーザーはいずれも、オフライン データベースに格納されている Microsoft Dynamics CRM データにアクセスできます。Outlook 用 CRM を実行するコンピューターの共用はサポートされていません。

オフラインになっているかどうかを見分けるには

  • Microsoft Dynamics CRM のツール バーに、[オンラインにする] ボタンが表示されている。

  • Microsoft Dynamics CRM のメニュー バーに、[オンラインにする] オプションが表示されている。

  • Outlook 2010 または Outlook 2013がインストールされている場合、[CRM] タブに、[オンラインにする] ボタンが表示されます。

Microsoft Dynamics CRM では、オフライン時のすべての追加および変更が、それを入力または更新したときと同じ順序で Microsoft Dynamics CRM サーバーに適用されます。オフライン時にレコードを変更している間に、別のユーザーが同じレコードを変更した場合は、オンラインに戻ったときに、オフライン時に別のユーザーによって行われた変更を上書きすることになります。

ワークフロー

オフライン作業中は、再びオンライン接続するまで、ワークフロー通知が生成されません。

また、オフライン アカウントに加えた変更が、自動ワークフロー プロセスまたはシステム ジョブを呼び出すよう設定されている場合、そのプロセスは、再びオンライン接続した後に自動的に呼び出されます。

重複データ検出

オフライン作業中は、重複データ検出は行われません。再びオンライン接続したとき、または同期したときに、重複データ検出が自動的に呼び出されます。個人用オプションを設定し、重複が検出されたときに重複データを作成するかどうかを指定できます。

オフライン時のセキュリティのアクセス許可

オフライン時に与えられるセキュリティのアクセス許可と権利は、オンライン時と同じです。

フィルターの使用

オフラインにする前に、フィルターによって Microsoft Dynamics CRM データの一部を選択して、ローカル コンピューターで使用できるようにします。オフライン作業中にこのデータに加えた変更は、データのローカル コピー上に保存されます。

これらのフィルターは、オフラインにするたびにリセットできるため、必要なデータだけをダウンロードできます。ローカル コピーから大量のデータを削除した場合は、同期に時間がかかる可能性があります。

オフラインで利用するデータは、フィルターを使用して定義します。フィルターは、ビジネス ニーズに合わせて編集できます。

  1. このタスクの実行には、既定のセキュリティ ロールすべてに含まれるアクセス許可が必要です。

  2. Outlook 2013 または Outlook 2010 がインストールされている場合は、[ファイル] をクリックまたはタップします。[CRM] タブをクリックまたはタップします。[オフラインにする] をクリックまたはタップしてから、[オフライン フィルター] をクリックまたはタップします。

    Outlook 2007 がインストールされている場合は、[CRM] メニューで [オフライン フィルター] をクリックまたはタップします。

    既定では、[オフライン同期フィルター] が表示されます。

  3. [ユーザー フィルター] タブで、編集するフィルターをクリックまたはタップします。新しいフィルターを作成するには、[操作] ツール バーで、[新規] をクリックまたはタップします。

    [新しいフィルター] ダイアログ ボックスにフィルターの名前と条件が一覧表示されます。このダイアログ ボックスを使用して、名前や説明などフィルターのプロパティを編集できます。また、フィルターの条件を追加または変更できます。たとえば、「所有者に Gabriela を含める」という条件を追加できます。

    各行はフィルター条件 1 セットを表し、3 つの列で構成されています。これらの列は、それぞれフィルターで使用するフィールド (取引先企業名 や 市区町村など)、クエリ関係演算子 (が次の値と等しいやが次の値を含むなど)、およびフィルター処理する値 ("オープン" など) に対応しています。

  4. [新しいフィルター] ダイアログ ボックスで条件を変更します。

    1. フィルター条件行を編集するには、フィールドをクリックまたはタップして別のフィールドを選択するか、クエリ関係演算子をクリックまたはタップして別の演算子を選択するか、または値をクリックまたはタップして新しい値を入力します。一部の値については、[このフィールドの値を選択または変更します] ボタン をクリックまたはタップして [値の選択] ダイアログ ボックスをオープンし、目的の値を選択できます。

    2. 条件行を追加するには

      1. [選択] をクリックまたはタップして、フィルター処理するフィールドを指定します。

      2. クエリ関係演算子をクリックして、演算子を選択します。

      3. [値の入力] をクリックまたはタップし、フィルター処理する値を入力します。一部の値については、[このフィールドの値を選択または変更します] ボタン をクリックまたはタップして [値の選択] ダイアログ ボックスをオープンし、目的の値を選択できます。

    3. 条件をグループ化するには、レコードの種類が同じである 2 つ以上の行を選択する必要があります。たとえば、[営業段階] と [売上見込み] は、どちらもレコードの種類が [営業案件] のフィールド値であるため、これらのフィールドのフィルター条件を指定する 2 つの行はグループ化することができます。ただし、[取引先企業] と [営業案件] のように、レコードの種類が異なるフィールド値を持つ行はグループ化できません。

      • グループ化する各行について、詳細モードで、その行の [オプション メニュー] ボタン をクリックまたはタップし、次に [行の選択] をクリックまたはタップします。

      • [フィルター] ツール バーで、[グループ AND] または [グループ OR] を選択します。

      • 行をグループから削除するには、その行の [オプション メニュー] ボタン をクリックまたはタップし、次に [削除] をクリックまたはタップします。

      • グループを選択するには、そのグループの [オプション メニュー] ボタン をクリックまたはタップし、次に [グループの選択] をクリックまたはタップします。

      • グループにフィルター条件句を追加するには、そのグループの [オプション メニュー] ボタン をクリックまたはタップし、[句の追加] をクリックまたはタップして、フィールド、クエリ関係演算子、および値を選択します。

      • 既に選択されているグループを選択解除するには、そのグループの [オプション メニュー] ボタン をクリックまたはタップし、次に [グループの選択解除] をクリックまたはタップします。

      • グループを選択解除するには、そのグループの [オプション メニュー] ボタン をクリックまたはタップし、[グループ化の解除] をクリックまたはタップします。

      • [グループ AND] グループを [グループ OR] グループに、または [グループ OR] グループを [グループ AND] グループに変更するには、[オプション メニュー] ボタン をクリックまたはタップし、次に [OR に変更] または [AND に変更] をクリックまたはタップします。

ヒント

  • すべての条件をクリアしてもう一度やり直すには、[フィルター] ツール バーで [クリア] をクリックまたはタップし、次に [確認] をクリックまたはタップします。

  • 行を削除するには、その行の [オプション メニュー] ボタン をクリックまたはタップし、次に [削除] をクリックまたはタップします。

  • 詳細モードでの作業中に、簡易モードで行を表示しないようにするには、その行の [オプション メニュー] ボタン をクリックまたはタップし、[簡易モードで表示しない] をクリックまたはタップします。

  • 簡易モードでフィルター条件を表示したときに表示されない行があって、その行が再び表示されるようにする場合は、詳細モードでその行の [オプション メニュー] ボタン をクリックまたはタップし、[簡易モードで表示] をクリックまたはタップします。

Microsoft Office Outlook 用 Microsoft Dynamics CRM のプライバシー情報

Microsoft Office Outlook 用 Microsoft Dynamics CRM を使用するには、資格情報 (電子メール アドレスとパスワード) を使用してサインインする必要があります。こうした資格情報は、Microsoft Outlook を開くたびに入力しなくても済むように、ローカルに保存することができます。この情報の保存を選択した場合は、Outlook を開くたびに Outlook 用 CRM が自動的に Microsoft Dynamics CRM に接続されます。
初めて Outlook 用 CRM にサインインして使用した後は、インターネット利用時はお客様のコンピューターと CRM が常に接続された状態になります。コンピューターと CRM との接続は、[オプション] 領域でオフにできます。ただし、このオプションを無効にすると、Outlook 用 CRM のパフォーマンスが低下する可能性があります。
Outlook 用 CRM を使用して、CRM の電子メールを追跡できます。その場合、電子メール スレッドはその閲覧のアクセス許可を持つ組織内のユーザーに表示されます。
Outlook 用 CRM は、お客様が受信するすべての電子メールについて、送信者の電子メール アドレス、受信者の電子メール アドレス、およびメッセージの件名を CRM に送信します。これにより、CRM では特定のメールを CRM サービスによって保存する必要があるかどうかを確認できます。アイテムが追跡対象になると、そのアイテムのコピーが CRM サービスによって保持され、適切なアクセス許可を持つ組織内の他のユーザーに表示されるようになります。アイテムの追跡を解除すると、そのアイテムを自分が所有している場合に限り、アイテムのコピーが CRM サービスから自動的に削除されます。
