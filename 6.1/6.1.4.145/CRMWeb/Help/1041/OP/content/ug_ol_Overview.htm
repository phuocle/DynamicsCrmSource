CRM の作業を Outlook 内で行う

Outlook 用 CRM は、デスクトップまたはラップトップ コンピューターにインストールされる Outlook 用のアドインです。Outlook 用 CRMを使用して、Microsoft Dynamics CRM Web アプリケーションで行える営業、マーケティング、および顧客サービスのほとんどを Outlook でも実行できます。これには、取引先企業、取引先担当者、およびキャンペーンの操作も含まれます。

Outlook 用 Microsoft Dynamics CRM と Microsoft Outlook での作業の違い

Microsoft Dynamics CRM では標準の Internet Explorer のコントロールを使用するため、Outlook の一部の機能は Outlook 用 CRM では利用できません。

Outlook 用 CRM で利用できない機能

回避策

電子メール署名

グローバル電子メール テンプレートを使用します。

フルテキスト編集機能

標準の編集コントロールを使用します。たとえば、ダブルスペースの行を入力するには Enter キーを押し、シングルスペースの行を入力するには Shift+Enterキーを押します。

作成中のファイルの添付

電子メール活動を保存した後にファイルを添付します。

ドラッグ アンド ドロップ操作によるレコードの移動

ありません。この操作を行うとデータが失われます。

アラーム

Outlook と同期した後は、Microsoft Dynamics CRM フォルダーまたは Web アプリケーションで作成した予定とサービス活動に対して 15 分のアラームが作成されます。

Outlook での Microsoft Dynamics CRM データのビューの操作

Microsoft Dynamics CRM レコードを Outlook で操作すると、タブにどんなレコードをどのように表示するかを変更できます。情報のフィルター処理や、レコードの並べ替えおよびグループ化が自由に行えます。

ビューは、情報がタブでどのように表示されるかを定義します。レコードが多数ある場合は、1 つのタブの複数のページにわたってそれらを表示できます。ビューは表示を固定することができ、場合によってはこの操作によってそうしたレコードがタブ上の複数のページから単一のリストに移されます。レコードのリスト ビューはコンピューターに保存されます。

一部のビューはサーバー専用です。つまり、システム管理者に定義された特定のビューは、コンピューターに保存できません。ビューの表示を固定することはできますが、レコードが単一のリストに移されることはありません。

オフライン作業

オフラインに移る前に、Outlook 用 CRM データのうちで、オフラインでローカル レコードとして使用するものを選択できます。再度オンラインに移るときに、ローカル レコードがサーバーと同期され、ローカル レコードに加えた変更が Microsoft Dynamics CRM データベースに反映されます。

詳細 : Outlook 用 CRM をオフラインで作業

重要

Outlook 用 Microsoft Dynamics CRM がインストールされているコンピューターにローカルにログオンするユーザーはいずれも、オフライン データベースに格納されている Microsoft Dynamics CRM データにアクセスできます。Outlook 用 CRM を実行するコンピューターの共用はサポートされていません。

Outlook 用 Microsoft Dynamics CRM および Microsoft Dynamics CRM の同期

Microsoft Dynamics CRM がオフラインの間は、Outlook 用 CRM を使用して、取引先担当者、タスク、予定、電話、レター、FAX、およびサービス予定について、作成、削除、および更新を行うことができます。オフライン中にこれらの変更を行うと、Outlook が更新されますが、Microsoft Dynamics CRM に反映されるのは、再度オンラインになった後です。

Word を使用した差し込み印刷文書の作成

Outlook 用 CRM を Microsoft Office Word と共に使用して、Microsoft Dynamics CRM データを表示する顧客仕様の文書を顧客向けに作成できます。

  • 新しいテンプレートを作成するか、既存のテンプレートを使用して、自分用のフォーム レターを生成します。適切なアクセス許可があれば、組織全体用のフォーム レターも生成できます。

  • FAX を作成します。

  • 見積もりを作成します。

複数のユーザーでのコンピューターの共有

業務上、複数のユーザーが同じコンピューターをそれぞれ異なるユーザー アカウントで使用する場合には、各ユーザーが自らのアカウントで Outlook 用 CRM をインストールおよび使用できます。その場合も、Outlook と Microsoft Dynamics CRM の通常の同期は行われます。

Microsoft Office Outlook 用 Microsoft Dynamics CRM のプライバシー情報

Microsoft Office Outlook 用 Microsoft Dynamics CRM を使用するには、資格情報 (電子メール アドレスとパスワード) を使用してサインインする必要があります。こうした資格情報は、Microsoft Outlook を開くたびに入力しなくても済むように、ローカルに保存することができます。この情報の保存を選択した場合は、Outlook を開くたびに Outlook 用 CRM が自動的に Microsoft Dynamics CRM に接続されます。
初めて Outlook 用 CRM にサインインして使用した後は、インターネット利用時はお客様のコンピューターと CRM が常に接続された状態になります。コンピューターと CRM との接続は、[オプション] 領域でオフにできます。ただし、このオプションを無効にすると、Outlook 用 CRM のパフォーマンスが低下する可能性があります。
Outlook 用 CRM を使用して、CRM の電子メールを追跡できます。その場合、電子メール スレッドはその閲覧のアクセス許可を持つ組織内のユーザーに表示されます。
Outlook 用 CRM は、お客様が受信するすべての電子メールについて、送信者の電子メール アドレス、受信者の電子メール アドレス、およびメッセージの件名を CRM に送信します。これにより、CRM では特定のメールを CRM サービスによって保存する必要があるかどうかを確認できます。アイテムが追跡対象になると、そのアイテムのコピーが CRM サービスによって保持され、適切なアクセス許可を持つ組織内の他のユーザーに表示されるようになります。アイテムの追跡を解除すると、そのアイテムを自分が所有している場合に限り、アイテムのコピーが CRM サービスから自動的に削除されます。
