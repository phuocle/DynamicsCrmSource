電子メール サーバー プロファイルの作成とメールボックスの追加

Microsoft Dynamics CRM の電子メール サーバー プロファイルには、電子メール サーバーとの接続および関連するメールボックスの電子メール処理のため、サーバー側の同期が使用する設定が格納されます。CRM 組織の電子メールを処理するには、少なくとも 1 つの電子メール サーバー プロファイルを構成する必要があります。組織の電子メール システムが複雑な場合は、複数の電子メール サーバー プロファイルの作成が必要な場合があります。

電子メール サーバー プロファイルの作成

  1. Microsoft Dynamics CRM のシステム管理者セキュリティ ロールまたは同等のアクセス許可を持っている必要があります。

    セキュリティ ロールの確認

    1. 「 ユーザー プロファイルの表示」の手順を実行してください。

    2. 適切なアクセス許可を持っていない場合は、システム管理者に問い合わせてください。

  2. 使用しているアプリケーションに対応した手順を実行してください。

    CRM Web アプリケーションを使用する場合

    1. ナビゲーション バーで、[Microsoft Dynamics CRM]、[設定] の順にクリックまたはタップします。

      ナビゲーション バーに [設定] と表示されます。

    2. [設定] > [電子メール アクセス構成] > [電子メール サーバー プロファイル] をクリックまたはタップします。

    Outlook 用 CRM を使用する場合

    1. ナビゲーション ウィンドウで、必要に応じて組織を展開し、[設定] > [システム] > [電子メール アクセス構成] > [電子メール サーバー プロファイル] をクリックまたはタップします。

  3. [新規] > [Exchange プロファイル]または[POP3-SMTP プロファイル]をクリックまたはタップします。

  4. Exchange 電子メール サーバー プロファイルで、以下の情報を指定します:

    フィールド

    内容

    全般

    名前

    プロファイルに分かりやすい名前を指定します。

    内容

    電子メール サーバー プロファイルの目標に関する短い説明を入力します。

    サーバーの場所の自動検出

    サーバーの場所の検出に自動検出サービスを使用する場合は、[はい] をクリックまたはタップします。これは、Exchange サーバーに対するプロファイルを作成する場合にのみ適用されます。いいえを設定する場合、電子メール サーバーを手動で指定する必要があります。

    メモ

    メールボックスのサーバー位置が変更されない場合、パフォーマンスが影響を受ける場合があるため、自動検出を使用しないことをお勧めします。

    受信サーバーの位置および送信サーバーの位置

    いいえをサーバーの場所の自動検出で選択した場合、受信および送信電子メール サーバーの場所を入力します。

    Exchange Server では、下記の例で説明されているように、Exchange Web サービス (EWS) の URL フォーマット <EWS/exchange.asmx> を使用します。

    https://autodiscover-red001.mail.microsoftonline.com/ews/exchange.asmx

    https://autodiscover-red002.mail.emea.microsoftonline.com/ews/exchange.asmx

    資格情報

    認証方法

    指定された電子メール サーバーに接続するときに認証する方法を選択します。以下のいずれかを選択できます。

    • ユーザーまたはキューで指定した資格情報。このオプションを選択すると、ユーザーまたはキューのメールボックスのレコードに指定される資格情報は、ユーザーまたはキューの電子メールを送信または受信するために使用されます。

      メモ

      資格情報が Microsoft Dynamics CRM で保護されるようにするには、SQL 暗号化を使用して、メールボックスに保存されている資格情報を暗号化します。

    • 電子メール サーバー プロファイルで指定した資格情報。このオプションを選択する場合、電子メール サーバー プロファイルで指定された資格情報が、このプロファイルに関連付けられたすべてのユーザーやキューのメールボックスの、電子メールを送信または受信するために使用されます。資格情報は、プロファイルに関連付けられたメールボックスの偽装または委任のアクセス許可が必要です。このオプションには、例えば、プロファイルに関連付けられたメールボックスのためのExchange偽装権限などを構成する、電子メール サーバーの構成が必要です。

      メモ

      資格情報が Microsoft Dynamics CRM で保護されるようにするには、サーバー側の同期を使用して電子メールを処理する場合は、SQL の暗号化を使用して、電子メール サーバー プロファイルに保存されている資格情報を暗号化します。

    • Windows 統合認証。このオプションは Exchange と SMTP 電子メール サーバーの種類にのみ適用されます。このオプションを選択する場合、Microsoft Dynamics CRM 非同期サービスを構成した資格情報が使用されます。

    一覧に含まれるそのほかのオプションは Exchange に適用されません。

    ユーザー名

    このプロファイルに関連付けられたすべてのユーザーやキューのメールボックスの、電子メールを送信または受信する電子メール サーバーへの接続に使用する、ユーザー名を入力します。このフィールドは 次の資格情報を使用して認証する が 電子メール サーバー プロファイルで指定した資格情報に設定されている場合にのみ使用でき、有効です。指定したユーザー名には、このプロファイルに関連付けられたユーザーおよびキューのメールボックスから電子メールを送受信するための、アクセス許可が必要です。

    メモ

    Microsoft Dynamics CRM に対して HTTP を使用している場合は、[ユーザー名] フィールドと [パスワード] フィールドが無効になります。オプションを有効にするには、展開プロパティ AllowCredentialsEntryViaNonSecureChannels の値を 1 に変更します。 構成設定について説明します

    パスワード

    このプロファイルに関連付けられたすべてのユーザーやキューのメールボックスの、電子メールを送信または受信する電子メール サーバーへの接続に使用するユーザー名と共に使用する、ユーザーのパスワードを指定します。パスワードは安全に保存されます。

    メモ

    Microsoft Dynamics CRM に対して HTTP を使用している場合は、[ユーザー名] フィールドと [パスワード] フィールドが無効になります。オプションを有効にするには、展開プロパティ AllowCredentialsEntryViaNonSecureChannels の値を 1 に変更します。 構成設定について説明します

    偽装を使用する

    他のメールボックスに代わって電子メールを送信または受信するために Exchange 偽装権限を構成している場合は、[はい] をクリックまたはタップします。他のメールボックスに代わって電子メールを送信または受信するための Exchange 委任権限を構成している場合は、いいえをクリックします。 以下のExchange 偽装の構成方法をご覧ください。

    送信に同じ設定を使用する

    受信および送信接続に同じ資格の設定を使用する場合は、はいをクリックするかタップします。

    詳細設定

    受信ポート

    このフィールドは、受信電子メールにアクセスする電子メール サーバーのポートを示します。このフィールドは、レコードを保存するときに自動的に設定されます。

    送信ポート

    このフィールドは、送信電子メールにアクセスする電子メール サーバーのポートを示します。このフィールドは、レコードを保存するときに自動的に設定されます。

    送信接続に SSL を使用する

    電子メール チャネルがセキュリティ保護されたチャネルにあり、SSL を電子メールの送信に使用する必要がある場合、[はい] をクリックまたはタップします。Microsoft Dynamics CRM (設置型)では、このフィールドは既定では [はい] に設定されますが、構成データベース設定 “AllowNonSSLEmail” によってこれを上書きし、値を False に設定することができます。 構成設定について以下をご覧ください。

    受信接続に SSL を使用する

    電子メール チャネルがセキュリティ保護されたチャネルにあり、SSL を電子メールの受信に使用する必要がある場合、[はい]をクリックまたはタップします。Microsoft Dynamics CRM (設置型)では、このフィールドは既定では [はい] に設定されますが、構成データベース設定 “AllowNonSSLEmail” によってこれを上書きし、値を False に設定することができます。

    受信認証プロトコルおよび送信認証プロトコル

    受信電子メールおよび送信電子メールの認証に使用するプロトコルを選択します。

    失敗した電子メールを配信不能フォルダーに移動する

    配送不可フォルダーに未配布の電子メールを使用するには、はいをクリックするかタップします。CRMで電子メール メッセージの追跡中に電子メール アクティビティのエラーが発生し、このオプションがはいにセットされている場合、電子メール メッセージは配送不可フォルダーに移動されます。このオプションは、Exchange の電子メール サーバー プロファイルでのみ使用できます。

    [追加設定]

    次の差出人からの電子メールを処理する

    日時を選択します。その日時以降に受信した電子メールは、このプロファイルに関連付けられたすべてのメールボックスに対し、サーバー側の同期によって処理されます。現在の日付より前の値をセットする場合、関連するすべての新しいメールボックスに変更が適用され、それより以前に処理された電子メールが取出されます。

    同時接続の最大数

    メールボックスごとにCRMが対応する電子メール サーバーに接続できる、同時接続の最大数を入力します。値を大きくするとExchangeへの並行呼び出しが増加してパフォーマンスが向上します。ExchangeにMicrosoft Dynamics CRMからの多量のコール数に起因するエラーが発生する場合、値を小さくします。このフィールドの既定値は 10 です。資格情報をメールボックスまたは電子メール サーバー プロファイルで指定したかどうかに基づき、最大数を、メールボックスまたは電子メール サーバー プロファイルごとに考慮します。

    最小ポーリング間隔 (分単位)

    この電子メール サーバー プロファイルと関連するメールボックスのために、最小ポーリング間隔を分単位で入力します。ポーリング間隔により、新しい電子メール メッセージのためにメールボックスにポーリングする、サーバー側の同期頻度が決定されます。

    内容

    受信サーバーの位置および送信サーバーの位置

    "<server>" 形式で、受信電子メール サーバーと送信電子メール サーバーの場所を入力します。

    • Windows 資格情報 (匿名)。匿名認証が電子メール サーバー上で有効になっている場合は、このオプションを選択できます。このオプションは SMTP 電子メール サーバーの種類にのみ適用されます。

電子メール サーバー プロファイルにメールボックスを追加

  1. Microsoft Dynamics CRM のシステム管理者セキュリティ ロールまたは同等のアクセス許可を持っている必要があります。

    セキュリティ ロールの確認

    1. 「 ユーザー プロファイルの表示」の手順を実行してください。

    2. 適切なアクセス許可を持っていない場合は、システム管理者に問い合わせてください。

  2. 使用しているアプリケーションに対応した手順を実行してください。

    CRM Web アプリケーションを使用する場合

    1. ナビゲーション バーで、[Microsoft Dynamics CRM]、[設定] の順にクリックまたはタップします。

      ナビゲーション バーに [設定] と表示されます。

    2. [設定] > [電子メール アクセス構成] > [メールボックス] をクリックまたはタップします。

    Outlook 用 CRM を使用する場合

    1. ナビゲーション ウィンドウで、必要に応じて組織を展開し、[設定] > [システム] > [電子メール アクセス構成] > [メールボックス] をクリックまたはタップします。

  3. 電子メール サーバー プロファイルに関連付けるすべてのメールボックスを選択し、[編集] をクリックまたはタップします。

  4. [複数のレコードを変更]フォームの [同期方法] の下にある [サーバー プロファイル] で、メールボックスを追加する電子メール サーバー プロファイルを選択し、次に 変更 をクリックまたはタップします。

  5. または、複数のメールボックスを選択し、より多くのコマンド (...)> [既定の電子メール設定の適用] をクリックまたはタップし、次に [OK] をクリックまたはタップします。メールボックスが、システムの設定ダイアログ ボックスで既定として選択した電子メール サーバー プロファイルに追加されます。 システムの設定ダイアログ ボックス - 電子メール タブの、既定の電子メール設定を参照してください。
