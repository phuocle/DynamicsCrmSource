フォーム フィールド プロパティの編集

  1. 使用しているアプリケーションに対応した手順を実行してください。

    CRM Web アプリケーションを使用する場合

    1. ナビゲーション バーで、[Microsoft Dynamics CRM]> [設定] をクリックまたはタップします。

      ナビゲーション バーに [設定] と表示されます。

    2. [設定] > [カスタマイズ] > [システムのカスタマイズ] をクリックするか、またはタップします。

    Outlook 用 CRM を使用する場合

    1. ナビゲーション ウィンドウで、[設定] > [カスタマイズ] > [カスタマイズ] > システムのカスタマイズをクリックまたはタップします。

  2. [コンポーネント] で、[エンティティ] を展開し、作業するエンティティを展開し、次に [フォーム] をクリックまたはタップします。

  3. ボックスの一覧で、[メイン] の [フォームの種類] のエントリを探してから、編集するためにダブルクリックするか、タップします。

  4. フォームの本文で、編集するフィールドをダブルクリックまたはタップします。

  5. [フィールドのプロパティ] ダイアログ ボックスで、一連のタブを使用して、フィールドに関連するさまざまプロパティにアクセスおよび編集できます:

    • 表示方法。このタブは、次のタスクを実行する場合に使用します。

      • このフィールドに使用されるのラベルを、このフォームで変更したり非表示にしたりします。フィールドの表示名は変更されません。

      • フィールドが、[フィールドは読み取り専用] チェック ボックスをオンまたはオフすることで、読み取り専用かどうかを指定します。

      • [ロック] セクションで、フィールドをフォームにロックするかどうか、[フォーム上でこのフィールドをロック] チェック ボックスをオンまたはオフにすることにより、だれによっても削除できないようにするかどうかを指定します。

      • フィールドが、[既定で表示される] チェック ボックスをオンまたはオフにすることによって、既定のビューに表示するかどうかを指定します。チェック ボックスをオフにすると、Microsoft Office Outlook 用 Microsoft Dynamics CRM の閲覧ウィンドウにフィールドが表示されません。しかし、JScript を使用して他の動作を定義していない限り、フィールドは Microsoft Dynamics CRM web アプリケーションに表示されます。web アプリケーションでは、このチェック ボックスによってJScript がその値を上書きします。

        検索フィールドの場合、このタブには追加プロパティが表示されます。

        • 検索に表示されたレコードの一覧をフィルターするには、[関連レコードのフィルタリング] セクションで [次の条件を満たすレコードのみ表示する] チェック ボックスをオンにします。

          最初の一覧には、この検索のフィルターに使用できるすべての関連付けが設定されています。いずれかをクリックしてください。

          すると、2 番目のリストに、関連エンティティ (最初のリストで選択済み) をターゲット エンティティに接続するすべての関連付けが設定されます。いずれかをクリックしてください。

          [ユーザーがフィルターを無効にできる] チェック ボックスをオンにして、ここで定義するフィルターをユーザーが無効にできるようにします。これにより、広い範囲のレコードを表示できます。このフィルターによって定義される、限られた範囲のレコードのみを表示できるようにするには、このチェック ボックスをオフにします。

        • また、検索フィールドの場合は、タブに [追加のプロパティ] セクションが表示されます。検索で検索ボックスを使用できるようにする場合は、このセクションで [検索ダイアログで検索ボックスを表示する] チェック ボックスをオンにします。

          [既定のビュー] ボックスの一覧で、検索に結果を表示する既定のビューをクリックします。

          ユーザーが他のビューも選択できるようにする場合は、[検索ダイアログでビュー選択を有効にする] チェック ボックスをオンにします。

    • 形式。表示される書式設定オプションは、どのオプションがフィールドが表示されるフォームのセクションに対して設定されているかに基づいて変更されます。

    • [詳細]。このタブでは、フィールドに関連する基本プロパティへのアクセスを提供します。変更するには [編集] をクリックまたはタップします。

    • イベント。このタブには 2 種類のセクションがあります:

      • [フォーム ライブラリ]。このセクションで、フォーム イベントまたはフィールド イベントに対して使用可能な JScript ライブラリを追加または削除します。必要に応じて、一覧表示されているユーザー定義ライブラリは編集できます。

      • [イベント ハンドラー]。このセクションでは、開発者が作成した JScript (スクリプト web リソース) を追加して、そのライブラリ内の関数をイベントに関連付けることができます。

      1. [イベント] の一覧で [onChange] をクリックまたはタップし、[追加] をクリックまたはタップします。

      2. [ハンドラーのプロパティ] ダイアログ ボックスで必要な情報を入力します。[有効] チェック ボックスをオンにして、この関数をフィールド イベントが呼び出せるようにします。

      3. [OK] をクリックまたはタップします。

    • 業務ルール。業務ルールのプロセス コンポーネントを作成または編集するには、このタブを使用します。

  6. フィールドのプロパティの変更が終了したら、[フィールドのプロパティ] ダイアログ ボックスで、[OK] をクリックまたはタップします。

  7. フォームがどのように表示されるか、およびイベントがどのように機能するかをプレビューするには、次のようにします。

    1. [ホーム] タブで [プレビュー] をクリックまたはタップし、[フォームの作成]、[フォームの更新]、または [読み取り専用] を選択します。

    2. プレビュー フォームを閉じるには、ファイル メニューの 閉じる をクリックまたはタップします。

  8. フォームの編集が終了したら、ホームタブで、[保存して閉じる] をクリックまたはタップして、フォームを閉じます。

  9. カスタマイズが完了したら、公開します。

    • 現在編集中のコンポーネントのみのカスタマイズを公開するには、ナビゲーション バーまたはナビゲーション ウィンドウで、取り組んでいるエンティティをクリックまたはタップし、その後、[パブリッシュ] をクリックまたはタップします。

    • 非公開のすべてのコンポーネントのカスタマイズを一度に公開するには、ナビゲーション ウインドウで、[エンティティ] をクリックまたはタップし、その後、[操作] ツール バーで、[すべてのカスタマイズの公開] をクリックまたはタップします。

メモ

ソリューションのインストールやカスタマイズの発行は、通常のシステム操作に影響を与える可能性があります。ソリューションのインポートは、ユーザーへの影響が最小限に留まるようにスケジューリングすることを推奨します。
