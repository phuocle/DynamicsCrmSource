レポートのカスタマイズおよび整理

レポートを使用して Microsoft Dynamics CRM のデータを分析します。Microsoft Dynamics CRM には、一般的な多くのビジネス ニーズに適合する既定のレポートが用意されています。しかし、ほとんどの組織では、特定のニーズに合わせて既定のレポートをカスタマイズしたり、カスタム レポートを追加したりしています。

レポートの所有者

システム レポートは、すべてのユーザーが使用できます。ユーザーが所有するレポートは、特定の同僚またはチームと共有することや、すべてのユーザーがレポートを使用できるように、組織に対してそのレポートを使用可能にすることができます。

レポートの種類

Microsoft Dynamics CRM は以下の 2 種類のレポートをサポートします。

  • Microsoft SQL Server Reporting Servicesレポート。これらのレポートは SQL クエリとフィルター ビューを使用してレポート データを取得します。フィルター ビューは、レポートを実行しているユーザーのセキュリティ ロールで使用できるものにデータを制限します。Microsoft Dynamics CRM に付属する既定のレポートはすべて、SQL ベースのレポートです。

    SQL データベースにアクセスする機能をサポートしていないため、Microsoft Dynamics CRM Online のフィルター ビューにアクセスすることはできません。したがって Microsoft Dynamics CRM Online サブスクリプションがある場合は、カスタム レポートにフェッチ ベースのレポートを使用します。

  • フェッチベースの Reporting Services レポート。これらのレポートは、フィルター ビューではなく Microsoft Dynamics CRM 特有の FetchXML クエリを使用し、レポート データを取得します。Microsoft Dynamics CRM のレポート ウィザードを使用して作成したレポートは、フェッチ ベースのレポートです。

他のレポートは、次のようなものです。

  • Web ページへのリンク

  • 静的ファイル

  • Microsoft Dynamics CRM データベースからデータを読み込む動的な Microsoft Office Excel ファイル

各レポートについて、以下のプロパティを編集できます。

  • ファイル名または URL

  • 表示名

  • 内容

  • ユーザー インターフェイスでレポートが表示される場所に関する情報

レポートのデータのセキュリティ

すべてのレポートは、フィルタされたビューから Microsoft Dynamics CRM データを読み取ります。フィルタされたビューでは、ユーザーのセキュリティ ロールに基づいてデータがフィルタリングされます。レポートには、そのレポートを実行しているユーザーが表示のアクセス許可を持っているデータのみが表示されます。

新規レポートを作成するオプション

適切なアクセス許可を持つユーザーは、新規レポートを作成するために、以下の操作を実行できます。

  • ファイルまたは Web ページへのリンクをレポートとして追加します。

  • レポート ウィザードを実行して、新しい Reporting Services レポートを作成します。レポート ウィザードでは、テーブル レポートおよびグラフ レポートが作成されます。これには、ドリルスルー レポートおよび上位 N 件レポートが含まれます。

  • Fetch ベースの Reporting Services の新規レポートを作成します。カスタムのフェッチ ベースのレポートを作成するには、Microsoft Dynamics CRM Report Authoring 拡張をインストールする必要があります。詳細については、『 Microsoft Dynamics CRM 2013 および Microsoft Dynamics CRM Online のレポート作成ガイド』の「フェッチ ベースの Reporting Services のカスタム レポートの作成」を参照してください。

既存のレポートを変更するためのオプション

既存のレポートに対して、適切なアクセス許可を持つユーザーは以下の操作を実行できます。

  • レポートをカテゴリに分類して、各レポートがレポート領域内で表示されるビューを制御する。

  • レポートを表示するユーザー インターフェイス内の場所を決定し、レポートのその他のプロパティを編集する。

  • レポート ウィザードで作成されたレポートを編集します。

  • 既定のレポートを編集する。たとえば、Microsoft Dynamics CRM をカスタマイズする場合は、既定のレポートのラベルの変更や、フィールドの追加または削除が必要になることがあります。詳細については、『 Microsoft Dynamics CRM 2013 および Microsoft Dynamics CRM Online のレポート作成ガイド』、『 SQL Server Reporting Services を使用する Microsoft Dynamics CRM のレポートの作成』の「既定のレポートの編集または SQL ベースの Reporting Services レポートの作成」を参照してください。

  • 既定のレポート、レポート ウィザードで作成されたレポート、またはその他の Reporting Services レポートの既定のフィルターを編集する。

  • Reporting Services レポートのワンタイム スナップショットを作成するか、または設定した間隔で実行されるように Reporting Services レポートのスケジュールを設定する。

  • 個人用レポートを他のユーザーと共有したり、組織内のすべての人が使用できるように設定する。

  • レポートを公開して、Microsoft SharePoint やカスタム プログラムなどの外部アプリケーションで使用できるようにします。

ソリューションでのレポート

Microsoft Dynamics CRM では、レポートがソリューションに対応しています。コンポーネントとしてソリューションに追加されたレポートは、Microsoft Dynamics CRM の機能およびユーザー インターフェイスを拡張するソフトウェアの 1 つの単位になります。ソリューションに追加できるレポートは、組織所有のもの、または組織に表示できるものに限られます。

メモ

レポートが組織に表示できるかどうかの確認するには、レポートの一覧からレポートを選択し、[編集] をクリックまたはタップします。次に、[管理] タブで ［公開対象] が [組織] に設定されているかどうかを確認します。

レポートのスナップショットは、ソリューションの一部として追加、インポート、エクスポートできます。Microsoft Dynamics CRM では、レポート、サブレポート、レポート カテゴリ、レポート表示領域、およびレポート関連のレコードの種類がレポート セットのコンポーネントと見なされます。ソリューションの更新を非上書きモードでインポートする際に、カスタマイズされたレポート セットのコンポーネントが存在した場合、そのソリューションによるレポートの更新はすべて無視されます。
