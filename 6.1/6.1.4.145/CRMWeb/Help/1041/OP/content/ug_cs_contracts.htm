契約を使用した顧客のサポート条件の定義

Microsoft Dynamics CRM の契約を使用して、サービス レベル契約 (SLA) と顧客にサービスを提供する契約条件を維持します。契約を使用して、契約の期間、サービスを受けられるサポート案件数や時間数、利用可能な曜日と時間帯、およびサービスの権利などの情報を定義します。

契約の作成

新規の契約は契約テンプレートに基づいて作成できます。これらは、契約を実際に記述する前に、サービス単位などの一部の情報を定義します。契約は既存の取引先企業および取引先担当者に対してのみ作成できます。

請求が行われるまで、契約は "下書き" の状態になります。新規契約にはそれぞれ一意の ID が割り当てられ、契約が更新される場合を除いて別の契約にその ID を使用することはできません。契約を更新すると、元の契約に対応する ID が付いた下書きとして保存されます。

契約の請求およびアクティブ化

契約に少なくとも 1 つの契約品目と請求先住所が登録されたら、契約の状態に請求済みのマークを付けることができます。契約の状態を変更することは、その契約を承認したことを意味します。

Microsoft Dynamics CRM が Microsoft BackOffice 製品と統合されていない限り、Microsoft Dynamics CRM で自動的に請求書が顧客に送信されたり、請求書レコードが作成されたりすることはありません。

開始日が経過するまで、契約状態は "請求済み" です。その後、アクティブに変わります。状態が "請求済み" または "アクティブ" な契約を削除することはできません。

契約の更新

契約の終了日が過ぎたら、その契約は期限切れになり、その契約に関するサポート案件をオープンできなくなります。契約に関する新しいサポート案件をオープンするには、契約を更新します。

契約を更新すると、期限切れになった元の契約と同じ ID 番号で契約の下書きが作成されます。新しい "下書き" の契約に対し、契約品目の追加や修正などの変更を行うことができます。元の契約でキャンセルされた契約品目を含めることができます。元の契約が期限切れになったときには、更新された契約の請求を行い、アクティブにすることができます。

更新された契約には元の契約と同じ ID 番号が割り当てられ、契約の履歴に更新処理が反映されます。開始日は、既存の契約の期限が切れる日の翌日に自動的に設定されます。Microsoft Dynamics CRM では、更新された契約に元の契約と同じ期間が自動的に与えられますが、その期間は手動で変更できます。
