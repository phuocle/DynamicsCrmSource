場所を初めて作成または追加

Microsoft Dynamics CRM からレコードのドキュメントを管理するには、ドキュメントが保存される SharePoint 上のライブラリおよびフォルダーを指す場所レコードを Microsoft Dynamics CRM に作成する必要があります。

  1. 管理するドキュメントを含む Microsoft Dynamics CRM レコードを開きます。

  2. [関連] で、[ドキュメント] をクリックします。

  3. 場所が関連付けられていない場合、CRM では [ドキュメントの場所の追加] ダイアログ ボックスまたは [SharePoint の場所の作成] ダイアログ ボックスが表示されます。次の表に、考えられるシナリオを示します。表示内容に応じて、「次の手順」列の手順を実行してください。

表示

次の手順

SharePoint内のサイト コレクションを指すサイト レコードが少なくとも 1 つ Microsoft Dynamics CRM 内にあり、そのサイト レコードの [リスト コンポーネントがインストール済み] チェック ボックスがオンになっている。

[ドキュメントの場所の追加] が開き、絶対 URL または相対 URL を指定するオプションが表示されます。

  1. [名前] ボックスで、名前を確認または入力します。ドキュメントの場所レコードがこの名前で Microsoft Dynamics CRM に作成されます。

  2. [ドキュメントの場所] ボックスに、場所レコードが指す SharePoint フォルダーの URL を入力します。この場所は、ドキュメントを保存する Microsoft Dynamics CRM レコードに関連付けられます。

    - または -

    [新しいフォルダーの作成] をクリックします。[親サイトまたは場所] ボックスで [検索] ボタン をクリックして、SharePoint 上でフォルダーが作成される既存のドキュメントの場所レコードを選択します。[フォルダー名] ボックスで、名前を検証または変更し、[OK] をクリックします。この名前のフォルダーは、SharePoint で作成されます。

  3. [保存]  をクリックします。

Microsoft Dynamics CRM にサイト レコードが 1 つあるが、[リスト コンポーネントがインストール済み] チェック ボックスがオンになっていない。

絶対 URL オプションのみある [ドキュメントの場所の追加] が表示されます。

  1. [名前] ボックスで、名前を確認または入力します。ドキュメントの場所レコードがこの名前で Microsoft Dynamics CRM に作成されます。

  2. [ドキュメントの場所] ボックスに、場所レコードが指す SharePoint フォルダーの URL を入力します。この場所レコードは、ドキュメントを保存する Microsoft Dynamics CRM レコードに関連付けられます。

  3. [保存]  をクリックします。

レコードに関連付けられている既存の場所はなく、ドキュメント管理設定ウィザードで指定された URL は SharePoint Server 2010 または SharePoint Server 2013 上のサイト コレクションの URL で、このサイト レコードの [リスト コンポーネントがインストール済み] チェック ボックスがオンになっている。

[SharePoint の場所の作成] が表示されます。

Microsoft Dynamics CRM は、SharePoint にフォルダーを自動作成しようとします。フォルダーが正常に作成された場合は、Microsoft Dynamics CRM で [SharePoint の場所の作成] が開き、新しいフォルダーの URL が表示されます。Microsoft Dynamics CRM は、この新しいフォルダーの URL を含むドキュメントの場所レコードを作成し、Microsoft Dynamics CRM レコードに関連付けます。

[OK]  をクリックします。

SharePoint 内のフォルダーへのリンクを含む新しいドキュメントの場所レコードが、Microsoft Dynamics CRM に作成されます。関連付けた場所が、[ドキュメントの場所] リストに追加されます。
