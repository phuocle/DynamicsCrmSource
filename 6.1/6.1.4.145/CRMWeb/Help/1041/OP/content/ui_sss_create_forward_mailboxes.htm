転送用メールボックスの作成またはメールボックスの編集

既定では、ユーザーおよびキューが Microsoft Dynamics CRM で作成されると、それぞれのメールボックスのレコードも作成されます。それらのメールボックスのレコードには、電子メール アドレス、メールボックスの資格情報、および電子メール同期方法などの、電子メール サーバー上の個々のメールボックスに固有の情報が含まれます。ユーザーおよびキュー用のサーバー側の同期を使用して電子メール メッセージを処理するには、それぞれのメールボックスのレコードは Microsoft Dynamics CRM の電子メール サーバー プロファイルのレコードに関連付ける必要があります。

組織が転送用メールボックスを使用してサーバー側の同期を構成する場合、新しい転送用メールボックスのレコードを作成できます。転送用メールボックスは、電子メール システムの各ユーザーのメールボックスからサーバー側のルールに基づいて転送される電子メール メッセージを収集するボックスとして使用されます。転送用メールボックスは、サーバー側の同期専用にする必要があり、個人ユーザーの作業用メールボックスとして使用することはできません。メールボックスの [受信電子メールの同期方法] が [転送用メールボックス]に設定されているユーザーおよびキューに対する電子メール メッセージを処理するのに使用できます。サーバー側の同期を使用して電子メールを処理するには、転送用メールボックスのレコードを、電子メール サーバー プロファイルのレコードに関連付けられなければなりません。 転送用メールボックスの詳細については、を参照してください。

  1. システム管理者またはシステム カスタマイザーのセキュリティ ロール、または同等のアクセス許可があることを確認してください。

    セキュリティ ロールの確認

    1. 「 ユーザー プロファイルの表示」の手順を実行してください。

    2. 適切なアクセス許可を持っていない場合は、システム管理者に問い合わせてください。

  2. 使用しているアプリケーションに対応した手順を実行してください。

    CRM Web アプリケーションを使用する場合

    1. ナビゲーション バーで、[Microsoft Dynamics CRM]、[設定] の順にクリックまたはタップします。

      ナビゲーション バーに [設定] と表示されます。

    2. [設定] > [電子メール アクセス構成] > [メールボックス] をクリックまたはタップします。

    Outlook 用 CRM を使用する場合

    1. ナビゲーション ウィンドウで、必要に応じて組織を展開し、[設定] > [システム] > [電子メール アクセス構成] > [メールボックス] をクリックまたはタップします。

  3. [新しい転送用メールボックス] をクリックまたはタップします。または、既存のメールボックスのレコードを編集するには、メールボックスのレコードを開きます。

  4. メールボックスのレコードで、以下の情報を指定します。

    フィールド

    説明

    全般

    名前

    メールボックスに分かりやすい名前を入力します。

    所有者

    メールボックスの所有者を表示します。自動的に事前設定されたユーザーのメールボックスに対しては、メールボックスの所有者はユーザー自身です。自動的に事前設定されたキューのメールボックスに対しては、メールボックスの所有者はキューのレコードの所有者です。

    電子メール アドレス

    転送用メールボックスへの電子メール アドレスを、forwardmailbox@contoso.com のように入力します。

    ユーザーまたはキューのメールボックスに対しては、対応するユーザーまたはキューのレコード フォームで指定された電子メール アドレスと同じです。電子メール アドレスをここで編集すると、ユーザーまたはキューのレコードの電子メール アドレスは、自動的に更新されます。

    処理が終わった電子メールの削除

    処理後にメールボックスから電子メールを削除するかを指定します。このフィールドは、転送用メールボックスやキューのメールボックスに対してのみ使用可能で、[可能] に設定できます。

    関連

    メールボックスが関連付けられたユーザーまたはキューを選択します。このフィールドは空で、転送用メールボックス用に設定することはできません。

    転送用メールボックス

    このフィールドは、メールボックスのレコードが転送用メールボックスかどうかを示します。[不可]に設定すると、メールボックスのレコードが、Microsoft Dynamics CRMの個別のユーザーまたはキューに関連付けられていることを示します。

    資格情報

    電子メール処理での資格情報の使用を許可する

    このメールボックスに関連付けられた電子メール サーバーの [認証方法] が [ユーザーまたはキューで指定した資格情報] に設定されている場合、[可能] をクリックまたはタップします。このフィールドが [可能]に設定されている場合は、ユーザー名とパスワードを指定する必要があります。これらの資格情報は、電子メール サーバーのメールボックスから電子メールを送受信するのに使用されます。

    メモ

    資格情報が CRM で保護されることを確実にするには、サーバー側の同期を使用して電子メールを処理する場合は、SQL の暗号化を使用して、メールボックスに保存されている資格情報を暗号化します。

    同期方法

    サーバー プロファイル

    このメールボックスの電子メールの処理に使用する電子メール サーバー プロファイルを選択します。

    受信電子メール

    受信電子メールの配信方法を選択します。これは、受信電子メールがこのメールボックスへアクセスする方法を決定します。

    • ありません。電子メールは受信されません。

    • 転送用メールボックス。電子メールは転送用メールボックスを使用して受信されます。

    • Outlook 用 Microsoft Dynamics CRM.電子メールは Outlook 用 CRMを使用して受信されます。

    • サーバー側の同期または E-mail Router。電子メールはサーバー側の同期または E-mail Router を使用して受信されます。

    送信電子メール

    送信電子メールの配信方法を選択します。これは、送信電子メールがこのメールボックスへ送信される方法を決定します。

    • ありません。電子メールは送信されません。

    • Outlook 用 Microsoft Dynamics CRM.電子メールは Outlook 用 CRMを使用して受信されます。

    • サーバー側の同期または Email Router。電子メールはサーバー側の同期または Email Router を使用して送信されます。

    メモ

    転送用メールボックスでは、[なし] だけが許可されます。

    予定、取引先担当者、タスク

    CRM の予定、連絡先、およびタスクを同期するのに、Outlook 用 CRM またはサーバー側の同期のどちらを使用するかを選択します。

    [なし] を選択すると、予定、連絡先、およびタスクは同期されません。

    構成テストの結果

    受信電子メールの状態

    受信電子メール用の電子メール アクセス構成テストの結果を表示します。次のような状態があります:

    • 実行なし。電子メール アクセス構成テストは、このメールボックスには実行されていません。

    • 成功。受信電子メールは構成され、このメールボックスで電子メールは受信されます。

    • 失敗。受信電子メールは構成されていますが、対応している構成済みメールボックスから電子メールを取得することはできません。

    送信電子メールの状態

    送信電子メール用の電子メール アクセス構成テストの結果を表示します。次のような状態があります:

    • 実行なし。電子メール アクセス構成テストは、このメールボックスには実行されていません。

    • 成功。送信電子メールは構成され、電子メールはこのメールボックスから送信されます。

    • 失敗。送信電子メールは構成されていますが、対応している構成済みメールボックスから電子メールを送信することはできません。

    予定、取引先担当者、およびタスクの状態

    予定、取引先担当者、およびタスクの同期の結果を表示します。次のような状態があります:

    • 実行なし。同期は、このメールボックスについてはテストされていません。

    • 成功。このメールボックスについて、予定、連絡先、およびタスクを同期させることができます。

    • 失敗。このメールボックスについて、予定、連絡先、およびタスクを同期させることができません。

    メールボックスのテストの完了日時

    このフィールドには、電子メール構成がこのメールボックスのレコードに対してテストされた日付および時刻が表示されます。

  5. [保存] または [保存して&閉じる] をクリックまたはタップします。
