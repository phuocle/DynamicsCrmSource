SharePoint ドキュメントの Microsoft Dynamics CRM 内からの管理

ドキュメントを Microsoft SharePoint に保存し、SharePoint のドキュメント管理機能を使用して Microsoft Dynamics CRM 内からそれらを管理します。

一層効果的にアクセス、共有、共同作業するため、Microsoft Dynamics CRM を SharePoint と統合します。SharePoint に保存されたドキュメントは、不正なアクセスや使用から保護することができます。詳細: リスト、ライブラリ、フォルダー、ドキュメント、またはリスト アイテムのアクセス許可の管理

エンティティに対するドキュメント管理の有効化

エンティティのドキュメントを管理するには、該当するエンティティに対するドキュメント管理を有効にする必要があります。SharePointのすべてのエンティティについてドキュメントを管理できます。既定では、組織に対してドキュメント管理を有効にするとき、ドキュメント管理は次のエンティティに対して自動的に有効にされます。

ドキュメント管理を設定すると、SharePointにより、Microsoft Dynamics CRMエンティティとレコードのコンテキストに応じてドキュメント ライブラリ、およびフォルダーが作成されます。

ドキュメント管理の動作方法

ドキュメント管理を設定するとき、ドキュメントが保存される SharePoint のサイト コレクションまたはサイト URL を指定します。詳細 : サービス拠点およびサイト コレクションの概要

Microsoft Dynamics CRMでは、指定されたサイト コレクション、またはサイトを使用してSharePoint上にエンティティ用のドキュメント ライブラリ、およびレコードのフォルダーを作成します。

SharePoint にドキュメント ライブラリおよびフォルダーを自動作成すると、新規作成されたフォルダーを指す該当ドキュメントの場所レコードが Microsoft Dynamics CRM 上に作成されます。

自動作成の機能を使用したくない場合は、SharePoint上にドキュメント ライブラリとフォルダーを手動で作成できます。次に、SharePoint上のドキュメント ライブラリおよびフォルダーを指すドキュメントの場所レコードをMicrosoft Dynamics CRMに作成してから、そのドキュメントの場所レコードとエンティティ レコードを関連付けます。

ドキュメントを保存するフォルダー構造を配置後、すべてのドキュメント管理機能の使用を開始できます。

ドキュメント ライブラリとフォルダーが自動的に作成される方法

フォルダー構造を作成するには、次の 2 つの方法があります。

  • 手動で、ドキュメント ライブラリとフォルダーを Microsoft Dynamics CRM エンティティとレコードのコンテキストに作成します。次に、Microsoft Dynamics CRM レコードと SharePoint フォルダーを関連付けます。

  • Microsoft Dynamics CRMにより、SharePointにドキュメント ライブラリとフォルダーが自動的に作成されます。

指定した URL が次のいずれかの場合は、Microsoft Dynamics CRMにより、SharePoint にドキュメント ライブラリとフォルダーが自動的に作成されます。

  • Microsoft SharePoint Server 2010 または Microsoft SharePoint Server 2013にあり、Microsoft Dynamics CRM List コンポーネント がインストールされているサイト コレクション。

  • Microsoft SharePoint Server 2010 または Microsoft SharePoint Server 2013にあるサイト コレクションの下のサイトで、サイト コレクションには Microsoft Dynamics CRM List コンポーネント がインストールされています。

Microsoft Dynamics CRM が自動的にフォルダー作成することを選択する場合、フォルダーが既定 (一般的) 構造に基づいて、あるいはエンティティに基づいて作成されるかを指定する必要があります。

  • 既定の構造各レコード用のフォルダーが、エンティティの対応するドキュメント ライブラリの下に作成されます。これは既定のフォルダー構造です。

    たとえば、100 bikesと250 bikesという営業案件レコードがあります。これらは Margie's Travelの営業案件です。Microsoft Dynamics CRM は次のようなフォルダー構造を作成します:

    ../opportunity/100 Bikes.

    この構造で:

    • opportunity はドキュメント ライブラリです。

    • 100 bikes および 250 bikes は営業案件のレコード フォルダーです。

  • エンティティに基づく構造フォルダーは、主エンティティ (ドキュメント管理の対象となる) のフォルダー、およびレコードが、選択した関連エンティティのドキュメント ライブラリの下に置かれるように作成されます。この構造を使用してフォルダーを作成する場合は、エンティティに基づくチェック ボックスを選択してから、一覧 (たとえば取引先企業または取引先担当者) からエンティティを選択します。

    たとえば、100 bikesと250 bikesという営業案件レコードがあります。これらの営業案件は、所属取引先企業としてMargie's Travel に関連付けられます。エンティティの取引先企業を選択します。これが、作成されたフォルダー構造です。

    ../account/Margie's Travel/opportunity/100 bikes

    この構造で:

    • account はドキュメント ライブラリです。

    • Margie's Travel は、関連するレコードのフォルダーです。

    • opportunity は、営業案件レコードのフォルダーです。

    • 100 bikes と 250 bikes は、メイン エンティティの関連する営業案件レコード用のフォルダーです。

ドキュメント管理を設定すると、Microsoft Dynamics CRMにより、ドキュメント ライブラリが作成されます。ただし、Microsoft Dynamics CRM がドキュメント ライブラリに各種フォルダーを作成するのは、レコードの [ドキュメント] 一覧を開くときのみです。
